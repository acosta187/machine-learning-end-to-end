Ejecucin de una API con FastAPI usando Uvicorn

Este proyecto utiliza FastAPI como framework para crear una API REST y Uvicorn como servidor ASGI para ejecutarla.

Por qu es necesario Uvicorn?

FastAPI no es un servidor web.
FastAPI solo define: - Rutas (endpoints) - Lgica de negocio - Validacin de datos

Para que la API pueda: - Escuchar peticiones HTTP - Mantener conexiones activas - Responder a clientes (navegador, Python, curl, etc.)

se necesita un servidor ASGI, y el ms usado es Uvicorn.

Forma estndar de lanzar una API con FastAPI

La forma ms comn, recomendada y estndar de ejecutar una aplicacin FastAPI es:

uvicorn main:app --reload

Esta es la forma utilizada en: - Documentacin oficial de FastAPI - Tutoriales profesionales - Desarrollo local - Proyectos reales

Explicacin del comando

uvicorn main:app --reload

1. uvicorn

Es el servidor ASGI que ejecuta la aplicacin. Se encarga de: - Recibir solicitudes HTTP - Enrutarlas a FastAPI - Enviar las respuestas al cliente

2. main

Hace referencia al archivo main.py.

Ejemplo:

main.py

3. app

Es el objeto FastAPI definido dentro de main.py.

Ejemplo:

from fastapi import FastAPI

app = FastAPI()

4. main:app

Significa: Usa el objeto app que est dentro del archivo main.py

5. --reload

Activa la recarga automtica. Esto significa que: - Cada vez que se guarda un cambio en el cdigo - El servidor se reinicia automticamente

--reload solo debe usarse en desarrollo, no en produccin.

Ejemplo completo

Estructura del proyecto:

project/
 main.py

Contenido de main.py:

from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def root():
    return {"status": "API funcionando"}

Ejecucin:

uvicorn main:app --reload

La API quedar disponible en: - http://127.0.0.1:8000 - http://127.0.0.1:8000/docs (Swagger UI)

Forma estndar en produccin

En produccin, la forma recomendada es usar Gunicorn con workers de Uvicorn:

gunicorn main:app -k uvicorn.workers.UvicornWorker -w 4

Esto permite: - Manejar mltiples usuarios - Mejor rendimiento - Mayor estabilidad

Resumen

  • FastAPI define la API
  • Uvicorn ejecuta la API
  • main:app indica dnde est la aplicacin
  • --reload facilita el desarrollo
  • uvicorn main:app --reload es la forma estndar de ejecucin

Conclusin

Usar Uvicorn no es opcional: es la manera correcta y profesional de ejecutar una aplicacin FastAPI, tanto en desarrollo como en produccin.
